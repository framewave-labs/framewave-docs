## 1. 프로젝트 개요

A2팀의 생성형 AI 통합 엔진은 영상 제작 경험이나 나이에 관계없이 누구나 자신의 아이디어를 이미지와 짧은 영상으로 쉽게 구체화할 수 있도록 돕는 웹 기반 생성형 AI 영상 제작 서비스입니다.

이 서비스는 전문 제작자만을 위한 제작 보조 도구가 아니라, 복잡한 편집 프로그램이나 고급 프롬프트 작성법을 잘 몰라도 단계별 안내를 따라 배경, 캐릭터, 키이미지, 스토리보드 기반 영상 결과물을 만들고 다시 확인할 수 있는 흐름을 제공합니다.

핵심 방향은 다음과 같습니다.

```text
캐릭터/배경 이미지 생성 프롬프트 입력
→ 캐릭터/배경 이미지 후보 생성
→ 사용할 이미지 선택
→ 선택한 캐릭터/배경 이미지 기준 방향별 레퍼런스 확보
→ 필요 시 키이미지 생성
→ B서비스에서 받은 9개 컷이 3x3으로 들어있는 스토리보드 이미지 1장과 프롬프트 확보
→ B서비스 결과물 기반 영상 생성
→ 라이브러리 저장 및 조회
```

## 2. 문제 정의

최근 생성형 AI 도구를 활용하면 누구나 이미지나 영상을 만들 수 있지만, 실제로 처음 사용하는 사람에게는 여전히 진입 장벽이 높습니다. 영상 제작 도구는 편집 방식이 복잡하고, AI 생성 도구는 어떤 프롬프트를 입력해야 원하는 결과가 나오는지 알기 어렵습니다. 특히 디지털 도구에 익숙한 정도가 다른 사용자들이 같은 방식으로 접근하기에는 화면 구성, 생성 단계, 결과 저장 방식이 충분히 직관적이지 않은 경우가 많습니다.

따라서 A2팀 프로젝트는 특정 전문 직군만을 대상으로 하기보다, 다양한 연령과 경험 수준의 사용자가 웹에서 자신의 아이디어를 단계적으로 입력하고 결과를 확인하면서 쉽게 영상 제작 흐름을 따라갈 수 있는 서비스를 목표로 합니다. 단일 이미지 생성에 그치지 않고, 캐릭터/배경 이미지 후보를 생성한 뒤 선택한 이미지를 기준으로 방향별 레퍼런스를 확보하고, B서비스에서 받은 3x3 스토리보드 이미지와 프롬프트를 기반으로 영상 생성 흐름까지 확장합니다.

## 6. MVP 범위

### 6.1 2주차 Pass 범위

2주차 중간점검은 Pass / Fail 기준이므로, 다음 흐름이 실제 화면에서 끊기지 않고 동작하는 것을 최우선 목표로 합니다.

```text
프롬프트 입력
→ 캐릭터/배경 이미지 후보 생성
→ 결과 표시
→ 저장
→ 라이브러리 조회
```

| ID | 기능 | 설명 | 완료 기준 | 담당 |
| --- | --- | --- | --- | --- |
| M001 | 이미지 생성 프롬프트 입력 | 사용자가 캐릭터 또는 배경 이미지 생성용 프롬프트를 입력합니다. | 텍스트 입력 가능 | FE / UX |
| M002 | 이미지 생성 API 호출 | GPT Image 2.0 기반 이미지 생성을 요청합니다. | 테스트 프롬프트로 이미지 생성 성공 | BE |
| M003 | 이미지 결과 표시 | 생성된 캐릭터/배경 이미지 후보를 화면에 표시합니다. | 결과 영역에 이미지 표시 | FE |
| M004 | 이미지 저장 | 생성 이미지를 라이브러리에 저장합니다. | 저장 버튼 클릭 후 저장 상태 반영 | FE / BE |
| M005 | 이미지 라이브러리 조회 | 저장된 이미지를 다시 조회합니다. | 라이브러리 화면에서 저장 이미지 확인 | FE / BE |
| M006 | 기본 화면 설계 | 생성 화면, 결과 화면, 라이브러리 화면을 구성합니다. | 핵심 흐름을 이해할 수 있는 UI | UX / FE |

### 6.2 4주차 A등급 목표 범위

4주차 최종평가에서는 2주차 이미지 생성·저장 흐름 위에 방향별 레퍼런스 확보, 키이미지 생성, B서비스 결과물 기반 영상 생성 흐름을 확장합니다.

| ID | 기능 | 설명 | 완료 기준 | 담당 |
| --- | --- | --- | --- | --- |
| A001 | 캐릭터/배경 이미지 및 레퍼런스 확보 | 캐릭터/배경 이미지를 생성하고, 선택한 결과를 기준으로 방향별 레퍼런스 이미지를 확보합니다. | 이미지 생성 및 화면 표시 | FE / BE |
| A002 | 영상 생성 | B서비스에서 받은 9개 컷이 3x3으로 들어있는 스토리보드 이미지 1장과 프롬프트를 입력해 영상 결과를 생성합니다. | 영상 생성 request id 발급 | BE |
| A003 | 영상 상태 조회 | request id 기반으로 영상 생성 상태를 확인합니다. | queued, processing, completed, failed 상태 표시 | BE / FE |
| A004 | 영상 결과 표시 | completed 상태에서 3~15초 영상 미리보기를 표시합니다. | 영상 URL 또는 미리보기 확인 | FE / BE |
| A005 | 영상 재생성 | 입력된 스토리보드 이미지 또는 프롬프트 수정 후 다시 생성합니다. | 재생성 가능 | FE / BE |
| A006 | 미디어 라이브러리 | 이미지와 영상을 저장하고 다시 조회합니다. | 이미지/영상 카드 구분 표시 | FE / BE |
| A007 | 배포 | 프론트엔드는 Vercel, 백엔드는 AWS 기준으로 시연 가능한 상태를 만듭니다. | 데모 URL 접속 및 주요 흐름 확인 | FE / BE |

### 6.3 S등급 도전 범위

S등급 기능은 2주차 Pass와 4주차 A등급 핵심 기능이 안정적으로 완료된 후 진행합니다.

| ID | 기능 | 설명 | 완료 기준 | 담당 |
| --- | --- | --- | --- | --- |
| S001 | 역프롬프트 | 레퍼런스 이미지를 분석해 재현용 프롬프트를 추출합니다. | 이미지 업로드 후 프롬프트 추출 | BE / FE |
| S002 | 프롬프트 수정 | 추출된 프롬프트를 사용자가 수정합니다. | 수정 가능한 텍스트 영역 제공 | FE / UX |
| S003 | 이미지 재생성 | 수정된 프롬프트로 이미지를 다시 생성합니다. | 재생성 이미지 표시 | BE / FE |
| S004 | 재생성 결과 저장 | 재생성 이미지를 라이브러리에 저장합니다. | 라이브러리에서 재생성 결과 확인 | FE / BE |

```text

8. 저장 완료 상태가 표시됩니다.
9. 라이브러리 화면에서 저장된 이미지를 다시 확인합니다.
```

## 8. 화면 및 UX 요구사항

이 섹션은 PM 초안입니다. 실제 화면 구조, IA, 와이어프레임, UI 상태 정의는 UX/UI 담당자 확인 후 확정합니다.

| 화면 | PM 기준 요구사항 | 확정 필요 담당 |
| --- | --- | --- |
| 생성 화면 | 프롬프트 입력, 이미지/영상 생성 진입, 생성 버튼, 옵션 영역 필요 | UX/UI, FE |
| 결과 화면 | 생성 결과 이미지 또는 영상 미리보기 표시, 저장 버튼, 재생성 진입 필요 | UX/UI, FE |
| 라이브러리 화면 | 저장된 이미지와 영상을 카드 형태로 조회 | UX/UI, FE |
| 상세 화면 | 저장된 결과물의 프롬프트, 생성 타입, 생성 결과를 확인 | UX/UI, FE |
| 상태 화면 | 로딩, 실패, 빈 상태, 저장 완료 상태 제공 | UX/UI, FE |

## 10. API 요구사항

이 섹션은 프론트와 백엔드가 맞춰야 하는 기능 흐름만 정리합니다. 실제 endpoint, method, request body, response body, error code는 개발 담당자 협의 후 확정합니다.

| API 흐름 | 제품 기준 요구사항 | 확정 필요 |
| --- | --- | --- |
| 이미지 생성 요청 | 프롬프트를 입력하면 GPT Image 2.0 기반 이미지 생성 요청이 처리되어야 합니다. | endpoint, 요청 필드, 응답 포맷 |
| 이미지 저장 | 생성된 이미지를 라이브러리에 저장할 수 있어야 합니다. | 저장 API 구조, 저장 실패 처리 |
| 이미지 조회 | 라이브러리에서 저장된 이미지를 조회할 수 있어야 합니다. | 조회 API, 정렬/필터 여부 |
| 영상 생성 요청 | B서비스에서 받은 9개 컷이 3x3으로 들어있는 스토리보드 이미지 1장과 프롬프트 기반으로 영상 생성을 요청해야 합니다. | 영상 모델, 요청 구조, 파일 전달 방식 |
| 영상 상태 조회 | request id 기반으로 영상 생성 상태를 확인해야 합니다. | 상태값, polling 방식, 실패 응답 |
| 영상 저장 | 완성된 영상을 라이브러리에 저장할 수 있어야 합니다. | 저장 API 구조, 영상 URL 처리 |

### 10.1 에러 응답에서 고려할 상황

| 상황 | 사용자에게 필요한 안내 |
| --- | --- |
| 이미지 생성 실패 | 프롬프트를 수정하거나 다시 시도하도록 안내 |
| 영상 생성 실패 | 실패 상태와 재시도 가능 여부 안내 |
| 저장 실패 | 결과가 저장되지 않았음을 명확히 안내 |
| 라이브러리 조회 실패 | 잠시 후 다시 시도하도록 안내 |
| API 사용량 초과 | 생성 요청이 일시적으로 제한되었음을 안내 |
